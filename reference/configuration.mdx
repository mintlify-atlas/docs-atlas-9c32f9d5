title: "Configuring OpenClaude Portable with ai_settings.env"
description: "All OpenClaude Portable settings live in data/ai_settings.env. Learn how to read, edit, and reset this file to change providers, models, and API keys."
---

All configuration for OpenClaude Portable lives in a single file: `data/ai_settings.env`. This file is created automatically during first-run provider setup and updated whenever you run **Change Provider**. You can also edit it directly with any text editor.

## File location

```text
<your-drive>/data/ai_settings.env
```

## File format

The file uses simple `KEY=VALUE` pairs. Lines starting with `#` are comments and are ignored.

```bash data/ai_settings.env
# ========================================================
# Portable AI - Master Switchboard
# ========================================================
AI_PROVIDER=openai
CLAUDE_CODE_USE_OPENAI=1
OPENAI_API_KEY=sk-or-v1-xxxxxxxxxxxxx
OPENAI_BASE_URL=https://openrouter.ai/api/v1
OPENAI_MODEL=qwen/qwen-2.5-coder-32b-instruct:free
AI_DISPLAY_MODEL=qwen/qwen-2.5-coder-32b-instruct:free
```

## Changing your configuration

The recommended way is to run the Change Provider tool:

<CodeGroup>
```batch Windows
tools\Change_Provider.bat
```
```bash Linux / macOS
bash tools/change_provider.sh
```
</CodeGroup>

You can also edit `data/ai_settings.env` directly in any text editor. Changes take effect on the next launch.

## Resetting configuration

Delete `data/ai_settings.env` and restart the launcher. The first-run provider selection menu will appear again.

<Warning>
`data/ai_settings.env` contains your API key in plain text. Do not share this file or commit it to version control.
</Warning>
