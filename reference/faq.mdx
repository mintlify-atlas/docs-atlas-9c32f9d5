Answers to the most frequently asked questions about OpenClaude Portable.

<AccordionGroup>
  <Accordion title="Does it work without an internet connection?">
    Yes — if you use the Ollama provider. Set up Ollama with `tools/Setup_Local_Models.bat` (Windows) or `tools/setup_local_models.sh` (Linux/macOS) while connected to the internet to download the model. After that, launch with `--offline` to skip the update check and use Ollama for fully offline inference.

    Cloud providers (OpenRouter, NVIDIA NIM, Gemini, Claude, OpenAI, DeepSeek) always require an internet connection for API calls.
  </Accordion>

  <Accordion title="Does it leave anything on the host machine?">
    No. All data is redirected to the `data/` folder on your drive via `CLAUDE_CONFIG_DIR`, `XDG_CONFIG_HOME`, and `XDG_DATA_HOME`. No files are written to the host system's home directory, registry, or application data folders.
  </Accordion>

  <Accordion title="Is my API key safe?">
    Your API key is stored in `data/ai_settings.env` on your drive — not on the host machine. It is not transmitted anywhere except to your chosen AI provider for API calls. Keep the drive physically secure. Consider encrypting the USB drive if it contains sensitive information.
  </Accordion>

  <Accordion title="Can I switch AI providers after initial setup?">
    Yes. Run `tools\Change_Provider.bat` on Windows or `bash tools/change_provider.sh` on Linux/macOS at any time. You can also delete `data/ai_settings.env` and restart the launcher to trigger the first-run provider selection menu.
  </Accordion>

  <Accordion title="Can I use the same drive on Windows, Linux, and macOS?">
    Yes. The `data/` folder is cross-platform — session history, chat logs, API keys, and Ollama models are shared across all platforms. The launcher downloads the appropriate Node.js binary for each OS/architecture combination independently.
  </Accordion>

  <Accordion title="What is Limitless Mode and when should I use it?">
    Limitless Mode (`--dangerously-skip-permissions`) runs the agent without asking for confirmation before file writes or shell commands. Use it for well-defined, trusted tasks where you want maximum automation speed. Avoid it when working with sensitive files or when running unfamiliar prompts.
  </Accordion>

  <Accordion title="How do I update the OpenClaude engine?">
    The launcher checks for engine updates automatically once per day. If an update is available, it runs `npm install @gitlawb/openclaude@latest` before launching. You can also trigger a check manually by deleting `data/last_update_check.txt` and restarting the launcher. Use `--offline` to skip the update check.
  </Accordion>

  <Accordion title="What is the speed proxy and can I disable it?">
    The speed proxy (`tools/local-proxy.js`) sits between OpenClaude and Ollama and trims the large system prompt to ~300 tokens before each request, cutting first-token latency from 60–120 seconds to 5–20 seconds on CPU hardware. It only activates when using the Ollama provider. To disable it, remove or rename `tools/local-proxy.js` — the launcher skips the proxy if the file is not found.
  </Accordion>

  <Accordion title="How do I run the web dashboard?">
    Select option 3 from the main menu, or run `tools\Open_Dashboard.bat` (Windows) / `bash tools/open_dashboard.sh` (Linux/macOS). The dashboard starts at http://localhost:3000.
  </Accordion>

  <Accordion title="The dashboard port 3000 is already in use — what do I do?">
    The dashboard is already running from a previous session. Open http://localhost:3000 directly in your browser. If you need to restart it, close the existing dashboard process first.
  </Accordion>
</AccordionGroup>
