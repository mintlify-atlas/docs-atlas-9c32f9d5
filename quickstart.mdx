This page walks you through launching OpenClaude Portable for the first time — from copying the folder to your drive, through first-run setup, to selecting a provider and starting your first AI coding session.

<Note>
  First-time setup requires an internet connection to download Node.js (~25 MB) and the OpenClaude engine (~5 MB). After that, only API calls to your chosen provider need a connection — or none at all if you use Ollama in offline mode.
</Note>

## Launch steps

<Tabs>
  <Tab title="Windows">
    <Steps>
      <Step title="Copy the folder to your USB drive or any directory">
        Download and extract the OpenClaude Portable release. Place the `OpenClaude-Multi-Platform/` folder on your USB drive or any local directory. No installer or admin rights are required.
      </Step>
      <Step title="Run the Windows launcher">
        Open a terminal in the folder and run:

        ```batch
        .\START.bat
        ```

        You can also double-click `START.bat` in File Explorer.
      </Step>
      <Step title="Wait for first-run setup to complete">
        On first run, the launcher automatically downloads and installs:

        - **Node.js v22.14.0** (~25 MB) into `engine/node-win-x64/`
        - **OpenClaude engine** (~5 MB) into `engine/node_modules/`
        - **GitPortable** (required for Git Bash on Windows) into `engine/git-win-x64/`

        Subsequent launches skip this step entirely and go straight to the menu.
      </Step>
      <Step title="Choose your AI provider">
        On first run, the provider selection menu appears. Enter the number for your preferred provider (1–7) and follow the prompts to enter and verify your API key. Your key is saved to `data/ai_settings.env` on your drive.
      </Step>
      <Step title="Select Launch AI from the main menu">
        After provider setup completes, the main menu appears. Press `1` to launch the agent in Normal Mode, or wait 10 seconds for it to auto-launch.

        ```text
        1) Launch AI       — Normal Mode (Auto-starts in 10s)
        2) Limitless Mode  — Auto-executes everything (Advanced)
        3) Open Dashboard  — View your chats visually
        4) Change Provider — Switch your AI provider or API Key
        5) Setup Offline   — Download local AI models (Ollama)
        ```
      </Step>
    </Steps>
  </Tab>
  <Tab title="Linux / macOS">
    <Steps>
      <Step title="Copy the folder to any directory">
        Download and extract the OpenClaude Portable release to any directory on your machine or USB drive. No installer or elevated privileges are required.
      </Step>
      <Step title="Make the launcher executable">
        Open a terminal in the extracted folder and run:

        ```bash
        chmod +x start.sh
        ```

        This only needs to be done once.
      </Step>
      <Step title="Run the launcher">
        ```bash
        ./start.sh
        ```

        On first run, the launcher detects your OS and architecture (x64 or arm64), then automatically downloads and installs:

        - **Node.js v22.14.0** for your platform (~25 MB) into `engine/node-<platform>-<arch>/`
        - **OpenClaude engine** (~5 MB) into `engine/node_modules/`

        Subsequent launches skip this step entirely and go straight to the menu.
      </Step>
      <Step title="Choose your AI provider">
        On first run, the provider selection menu appears. Enter the number for your preferred provider (1–7) and follow the prompts to enter and verify your API key. Your key is saved to `data/ai_settings.env` on your drive.
      </Step>
      <Step title="Select Launch AI from the main menu">
        After provider setup completes, the main menu appears. Press `1` to launch the agent in Normal Mode, or wait 10 seconds for it to auto-launch.

        ```text
        1) Launch AI       — Normal Mode (Auto-starts in 10s)
        2) Limitless Mode  — Auto-executes everything (Advanced)
        3) Open Dashboard  — View your chats visually
        4) Change Provider — Switch your AI provider or API Key
        5) Setup Offline   — Download local AI models (Ollama)
        ```
      </Step>
    </Steps>
  </Tab>

## First run vs. repeat launches

| Scenario | What happens |
|---|---|
| **First run** | Downloads Node.js (~25 MB) and the OpenClaude engine (~5 MB), then prompts for provider and API key setup |
| **Repeat launch** | Skips download and setup; checks for engine updates once per day, then opens the main menu directly |
| **After `--offline` flag** | Skips the daily update check and opens the menu immediately |

## Main menu options

Each option in the main menu serves a distinct purpose:

| Option | Description |
|---|---|
| **1) Launch AI — Normal Mode** | Starts the agent in the terminal. The agent asks for approval before writing any file or running any shell command. Auto-launches after 10 seconds if no key is pressed. |
| **2) Limitless Mode** | Starts the agent in fully-autonomous mode. The agent executes file writes and shell commands without prompting for approval. |
| **3) Open Dashboard** | Starts the web dashboard server and opens `http://localhost:3000` in your browser — a ChatGPT-style chat interface with agent mode and tool cards. |
| **4) Change Provider** | Re-runs the provider selection menu so you can switch to a different AI provider or update your API key. |
| **5) Setup Offline** | Downloads a local Ollama model so you can run the agent with no internet connection after setup. |

## Launch flags

Pass flags directly to the launcher script to change startup behavior:

```bash
# Skip the daily update check and open the menu immediately
./start.sh --offline

# Skip the menu and auto-launch in Limitless Mode
./start.sh --quick
```

```batch
:: Windows equivalents
.\START.bat --offline
.\START.bat --quick
```

| Flag | Effect |
|---|---|
| `--offline` | Skips the engine update check. Useful on air-gapped networks or slow connections. |
| `--quick` | Bypasses the main menu and launches directly in Limitless Mode. |

<Warning>
  **Limitless Mode executes without approval.** In this mode the agent writes files and runs shell commands automatically, without asking first. Only use Limitless Mode on projects where you trust the agent to act autonomously.
</Warning>
